// Write-up file format

Each write-up lives in a writeups/ subfolder as its own HTML file. Recommended naming: YYYY-MM-CTFNAME-challenge-name.html

Suggested write-up structure:

  • Challenge overview — description, category, points, CTF event
  • Initial recon — what you observed first, tools used
  • Dead ends — approaches that didn't work and why (this is what separates a good write-up from a great one)
  • The solution — step-by-step exploit or methodology with code snippets
  • Key takeaway — one thing worth remembering
  • Flag — redact if required by the organizer
